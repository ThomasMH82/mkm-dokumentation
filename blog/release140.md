# Release 1.4

**Release Notes MKM Version 1.4** 

**Farbanpassung in der Speisekarte (Überschriften)** \
Die Überschriften in der Speisekarte wurden auf eine einheitliche schwarze Farbgebung umgestellt. Dadurch wird eine klare, konsistente Darstellung sichergestellt und die Lesbarkeit verbessert. 

**Kundenlogo in der Speisekarte einbinden können** \
Es ist jetzt möglich, ein eigenes Kundenlogo in der Speisekarte zu hinterlegen. Hierzu muss beim Onboarding einmalig das Kundenlogo hinterlegt werden. 

**Liefertage im Onboarding hinterlegen können** \
Bereits im Onboarding können feste Liefertage definiert werden. Bei der Bestellung werden diese automatisch in der Kalenderansicht berücksichtigt und vorgeschlagen. 

**Automatische Lieferantennummervergabe bei Anlage eines neuen Lieferanten** \
Beim Anlegen eines neuen Lieferanten wird eine Lieferantennummer nun automatisch vergeben. Das reduziert manuellen Aufwand und verhindert doppelte Nummern. 

**Alle Speisekarten sind nun dynamisch.** \
Speisekarten sind nicht mehr statisch, sondern passen sich automatisch an der Zeilenhöhe der Menülinien an. 

**Automatische Mengenoptimierung** \
Bei aktivierter Einstellung wird die benötigte Menge automatisch optimiert. Es ist nicht mehr notwendig, jeden einzelnen Tag in der Speisekarte manuell auszuwählen. 

**Preise immer mit €-Zeichen anzeigen** \
Alle Preise im System werden einheitlich mit dem €-Zeichen dargestellt. 

**Anpassung Bestellliste** \
In der Bestellliste wird jetzt angezeigt, in welcher Zeile man sich aktuell befindet. Zusätzlich wird die Produktverfügbarkeit direkt in der Liste dargestellt. 

**Hinterlegung von sekundären Produktkategorien** \
Produkte können nun zusätzlich zu ihrer Hauptkategorie einer oder mehreren sekundären Kategorien zugeordnet werden. 

**Speisekarten archivierbar machen können** \
Speisekarten können archiviert werden, ohne gelöscht zu werden. So bleiben sie weiterhin verfügbar. 

**Produkte ohne gültigen Preis bestellen können** \
Es ist nun möglich, Produkte auch dann zu bestellen, wenn kein gültiger Preis hinterlegt ist. Die Lieferanten geben im Anschluss Rückmeldung, ob das Produkt tatsächlich vorhanden ist. So werden Bestellungen bei kurzfristigen Preisänderungen nicht mehr verzögert. 

 **Änderung von Nährwerten verbieten** \
Die Bezeichnungen der Nährwerte sind nun fest definiert und können nur noch durch Super-Admins geändert werden.  

**Template Lieferantenbestellung anpassen** \
Das Template für Lieferantenbestellungen wurde leicht überarbeitet und an neue Anforderungen angepasst. 

**Neue Importvorlage für den Produktimport bei Fremdlieferanten** \
Für den Import von Produkten von Fremdlieferanten steht eine neue Importvorlage zur Verfügung. 

**Technische Anpassungen für den Sync und bei Äquivalenzen** \
Im Hintergrund wurden technische Optimierungen für die Synchronisation sowie für die Verarbeitung von Äquivalenzen vorgenommen, um Stabilität und Performance zu verbessern. 

**Link zur Schulungsdokumentation auf der Startseite einbinden** \
Auf der Startseite wurde ein direkter Link zur Schulungsdokumentation integriert, sodass Anwender schnell auf Hilfestellungen zugreifen können. 

**Anpassungen im Produktionsauftrag (Temperaturerfassung)** \
Die Temperaturerfassung im Produktionsauftrag wurde angepasst und verbessert. 

**Absenden leerer Bestellungen verhindern** \
Leere Bestellungen können zwar abgeschlossen werden, werden jedoch nicht mehr an die Lieferanten übermittelt.  

**Varianten in der Rezeptur nicht mehr bearbeitbar machen** \
Der Reiter „Varianten“ ist künftig nur noch lesend verfügbar. Das Anlegen und Bearbeiten von Varianten erfolgt ausschließlich über den Button **„Anpassung/Varianten erstellen“**, um Fehlbedienungen zu vermeiden. 

**Löschen von Äquivalenzzeilen** \
Beim Löschen von Äquivalenzzeilen wird nun geprüft, ob es sich um die letzte Zeile einer Äquivalenzgruppe handelt. Ist dies der Fall, wird die gesamte Gruppe automatisch entfernt. Zusätzlich werden bestehende leere Äquivalenzzeilen identifiziert und technisch bereinigt, um Inkonsistenzen zu vermeiden. 

**Wareneingangsbuchung – Spalte umbenennen** \
Die Spalte **„Erledigt“** wurde in **„Geliefert“** umbenannt, um die Bezeichnung klarer und verständlicher zu gestalten.
